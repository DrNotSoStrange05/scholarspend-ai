## 🔐 Security Considerations

When deploying to production:

1. **CORS**: Currently allows `"*"` (all origins)
   - In `backend/app/main.py`, update:
   ```python
   allow_origins=["https://your-flutter-app-domain"],  # Specify Flutter app only
   ```

2. **API Keys**: For SMS parsing, store securely
   - Use Railway secrets management
   - Never commit to GitHub

3. **Database**: PostgreSQL password auto-generated by cloud provider
   - Use strong credentials
   - Keep backups enabled

4. **HTTPS**: All cloud providers use HTTPS by default ✓

## 💡 How It Works

### Local Development
```
Flutter App (Device)
    ↓ HTTP
Local Backend (192.168.x.x:8000)
    ↓
SQLite/PostgreSQL
```

### Cloud Deployment
```
Flutter App (Anywhere)
    ↓ HTTPS
Railway Backend (global URL)
    ↓
PostgreSQL (managed by Railway)
```

## 🆘 If Something Goes Wrong

### Backend won't build
- Check Railway logs (Deployments tab)
- Verify `requirements.txt` has all dependencies
- Ensure Dockerfile syntax is correct

### Can't connect from Flutter
- Verify URL in `api_service.dart`
- Test URL in browser: `https://your-url/health`
- Check no typos in base URL
- Ensure `/api` suffix is present

### Database errors
- Verify PostgreSQL add-on is active in Railway
- Check `DATABASE_URL` environment variable
- Wait a few seconds after creating database
